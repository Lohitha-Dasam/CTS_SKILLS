# DevOps and CI/CD Notes

## 1. What is DevOps?

DevOps is a software development approach that combines Development (Dev) and Operations (Ops) to improve collaboration, automate workflows, and deliver software faster and more reliably.

### Goals of DevOps

- Faster software delivery
- Better collaboration
- Automation of repetitive tasks
- Improved software quality
- Continuous monitoring
- Faster issue resolution

---

## 2. Benefits of DevOps

- Faster releases
- Improved collaboration
- Better software quality
- Reduced deployment failures
- Increased productivity
- Continuous feedback
- Reliable deployments

---

## 3. Continuous Integration (CI)

Continuous Integration is the practice of automatically building and testing code whenever developers push changes to a shared repository.

### CI Process

Developer
→ Commit Code
→ Build
→ Automated Testing
→ Merge

### Benefits

- Early bug detection
- Faster development
- Better code quality
- Automated testing

---

## 4. Continuous Delivery (CD)

Continuous Delivery automatically prepares software for deployment after successful testing.

Deployment to production requires manual approval.

### Benefits

- Reliable releases
- Faster deployment
- Lower deployment risk
- Easy rollback

---

## 5. Continuous Deployment

Continuous Deployment automatically deploys every successful change directly to production without manual approval.

---

## 6. CI vs CD

| Continuous Integration | Continuous Delivery |
|------------------------|---------------------|
| Automatically builds code | Automatically prepares deployment |
| Focuses on integration | Focuses on release |
| Developers commit frequently | Software is always deployment-ready |
