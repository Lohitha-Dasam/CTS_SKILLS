# Generative AI Fundamentals Notes

## 1. What is Generative AI?

Generative AI (GenAI) is a type of Artificial Intelligence that creates new content such as text, code, images, audio, and videos by learning patterns from existing data.

Unlike traditional AI, which focuses on prediction or classification, Generative AI creates original outputs.

---

## 2. Generative AI vs Traditional AI

| Generative AI | Traditional AI |
|---------------|----------------|
| Creates new content | Predicts or classifies data |
| Generates text, code, images | Detects patterns |
| Uses Large Language Models | Uses Machine Learning models |
| Creative tasks | Analytical tasks |

---

## 5. Prompt Engineering

Prompt Engineering is the process of writing effective prompts to obtain accurate and useful responses from AI models.

Good prompts improve output quality.

---

## 6. Prompting Techniques

### Zero-shot Prompting

The model is asked to perform a task without examples.

Example:

```
Write a Python function to reverse a string.
```

---

### Few-shot Prompting

The prompt includes one or more examples before asking the model to perform a task.

Example:

```
Input: 2 + 3
Output: 5

Input: 10 + 20
Output:
```

---

### Chain-of-Thought Prompting

The model is encouraged to reason step by step before producing the final answer.

Example:

```
Solve the problem step by step.
```

---

## 7. Prompt Engineering Best Practices

- Give clear instructions.
- Provide sufficient context.
- Specify the required output format.
- Use examples when needed.
- Refine prompts based on responses.
- Avoid ambiguous wording.

---

## 8. Ethical Considerations

Responsible AI usage includes:

- Avoid biased prompts.
- Verify AI-generated information.
- Protect sensitive data.
- Respect privacy.
- Use AI responsibly.
- Avoid plagiarism.

## 9. GitHub Copilot

GitHub Copilot is an AI-powered coding assistant developed by GitHub and OpenAI.

It helps developers write code faster by suggesting code completions and entire functions.

---

## 10. Features of GitHub Copilot

- Code Completion
- Function Generation
- Documentation Generation
- Test Case Generation
- Code Refactoring
- Boilerplate Code Generation
- Intelligent Suggestions

## 13. Setting up GitHub Copilot

Steps:

1. Install Visual Studio Code.
2. Install the GitHub Copilot extension.
3. Sign in using a GitHub account.
4. Enable Copilot.
5. Start writing code and accept suggestions using **Tab**.

---

## 14. Core Capabilities

GitHub Copilot can:

- Generate complete functions
- Suggest code snippets
- Write comments
- Generate documentation
- Create unit tests

---

## 16. Security Risks

AI-generated code may contain:

- Security vulnerabilities
- Incorrect implementations
- Hallucinated code
- Outdated APIs

Always review generated code before using it.

---

## 17. Licensing and Privacy

Considerations:

- Generated code may resemble public repositories.
- Check licensing requirements.
- Avoid sending confidential code unnecessarily.
- Follow your organization's security policies.

---

## 18. Responsible AI Usage

Best Practices:

- Review every AI suggestion.
- Test generated code thoroughly.
- Protect sensitive information.
- Follow secure coding standards.
- Use AI as an assistant, not a replacement for developer judgment.

---

## Summary

Generative AI enhances software development by assisting with code generation, documentation, testing, and refactoring. Prompt Engineering and GitHub Copilot improve developer productivity, while responsible AI practices ensure secure, ethical, and reliable use of AI-generated content.
